# Hands-on Labs

This folder is for practical projects that prove you can build, deploy, observe, and secure AI systems.

Do not treat AI learning as only reading. Every major concept should become a lab, demo, script, architecture diagram, or deployed service.

## Files

- [lab-backlog.md](lab-backlog.md): beginner, intermediate, and advanced lab ideas.
- [lab-template.md](lab-template.md): reusable template for each new lab.

## Lab Categories

- Beginner: LLM API calls, prompts, embeddings, simple search.
- Intermediate: RAG, agents, MCP, observability, evals.
- Advanced: Bedrock AgentCore, multi-agent SRE assistant, prompt-injection defense, model routing.

## Resume Goal

Each completed lab should produce:

- Working code.
- README.
- Architecture diagram.
- Tests or evals.
- Security notes.
- Observability notes.
- Resume bullet draft.
