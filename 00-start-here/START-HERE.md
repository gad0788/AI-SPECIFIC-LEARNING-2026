# Start Here

You are approaching AI from a strong DevOps/SRE/cloud base. That is an advantage because production AI is still distributed systems, APIs, automation, security, data pipelines, observability, and incident response.

## First 30 Days

Focus on vocabulary and building small things.

- Learn core terms: model, token, context window, embedding, vector search, RAG, agent, tool call, MCP, eval, guardrail, hallucination.
- Use Codex and Claude Code daily for real tasks, but always review the output like a pull request.
- Build one minimal LLM app using an API.
- Build one small RAG app over your own markdown notes.
- Configure at least one MCP server and understand what permissions it grants.
- Start a learning log and resume evidence bank.

## First 60 Days

Focus on production patterns.

- Build a RAG API with ingestion, chunking, embeddings, vector DB, retrieval, generation, logging, and tests.
- Add traces, token/cost logging, latency metrics, and eval datasets.
- Compare OpenAI, Anthropic, AWS Bedrock, Azure AI Foundry, and Google Vertex/Gemini at a high level.
- Learn agent patterns: ReAct, planner-executor, reflection, tool router, multi-agent handoff, human approval.
- Learn security risks: prompt injection, tool over-permission, data exfiltration, insecure MCP servers, untrusted retrieval content.

## First 90 Days

Focus on portfolio and resume impact.

- Deploy one AI app on AWS.
- Deploy one agent workflow with safe tool calling and audit logs.
- Build an observability dashboard for LLM calls and agent tool traces.
- Publish clean GitHub repos with README, architecture diagram, runbook, tests, and screenshots.
- Translate each project into ATS-ready resume bullets.

## Daily Study Loop

1. Learn: read docs or watch one focused lesson.
2. Build: apply it in a small repo or lab.
3. Operate: add logging, metrics, tests, security notes, and cost awareness.
4. Explain: write a short note in this folder.
5. Resume: add one bullet only when you can prove the work.
